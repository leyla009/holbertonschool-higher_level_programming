# Python - Exceptions

## Description
This project focuses on handling errors and exceptions in Python. It covers how to use `try`, `except`, `else`, and `finally` blocks to create robust code that doesn't crash when encountering unexpected input or states.

## Learning Objectives
* Why basic Python programming is awesome
* What’s the difference between errors and exceptions
* What are exceptions and how to use them
* When do we need to use exceptions
* How to correctly handle an exception
* What’s the purpose of catching exceptions
* How to raise a built-in exception
* When to implement a clean-up action after an exception

## Tasks

| File | Description |
| --- | --- |
| [0-safe_print_list.py](./0-safe_print_list.py) | Prints `x` elements of a list safely using `try: / except:`. |
| [1-safe_print_integer.py](./1-safe_print_integer.py) | Prints an integer with `"{:d}".format()` safely. |
| [2-safe_print_list_integers.py](./2-safe_print_list_integers.py) | Prints the first `x` integers of a list, skipping non-integers. |
| [3-safe_print_division.py](./3-safe_print_division.py) | Divides 2 integers and prints the result using `finally`. |
| [4-list_division.py](./4-list_division.py) | Divides element by element 2 lists and creates a new list. |
| [5-raise_exception.py](./5-raise_exception.py) | Raises a type exception. |
| [6-raise_exception_msg.py](./6-raise_exception_msg.py) | Raises a name exception with a message. |
